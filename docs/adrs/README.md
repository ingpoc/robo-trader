# ADRs

Use this directory for repo-local architecture decisions that shape Robo Trader implementation.

## When To Add One

- a decision changes repo structure or ownership
- a workflow rule affects multiple teams or future contributors
- a new integration changes auth, MCP, or system boundaries
- the repo needs a durable local fallback while Notion is unavailable

## Minimum Template

- Title
- Status
- Date
- Context
- Decision
- Consequences
- Links
  - Linear issue
  - Notion page
  - PR or commit

## Supersession

When an ADR replaces an older note, update the older file with a superseded pointer.
